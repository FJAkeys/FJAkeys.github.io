Common Lisp的环境配置

Posted on 2017-07-21

声明：

  • 我们以后都会使用common-lisp作为教程的使用lisp语言。为什么是common lisp而不是其他的lisp方言？因为cl有更好的标准和规范，适用环境更普遍，基础语法更简洁。
  • 这里包含三种典型的环境配置来满足你不同的需求，请先大致浏览整篇文章，再选择你自己适合的实现方式。
  • 本节的安装只针对linux平台(ubuntu)的安装，系统基本的操作我就不啰嗦了。没有用过linux的朋友建议尝试一下，挚友MattRow曾经告诉我最快学习linux的方式就是把你的windows覆盖掉，生活在linux中。感谢他的教导。

lispbox，cl环境一种最简便的解决方式

你完全可以修建一个属于自己的游乐场，比如用Emacs作为文本编辑器，SLIME(The Superior Lisp Interaction Mode for Emacs )作为Emacs的common-lisp连接模块(emacs本身只支持emacs-lisp)，quicklisp作为库管理器，以及SBCL作为语言的基本实现来构建。

是不是只是列举起来就很麻烦？

值得庆幸的是初期作为练习环境，lispbox给我们提供的方便的集成环境。你可以在官网上选取操作系统对应的版本进行下载，然后解压后在文件目录中找到名为lispbox.sh的执行文件，运行之后就可以看到emacs编辑器了。Emacs是一个用lisp实现的编辑器，自然也可以用lisp来进行扩展做任何你想做的事(当然是基于EmacsLisp的)，比如连接咖啡机帮你煮咖啡。在你打开emacs编辑器之后，可以按下Ctrl+h，再按一下t，emacs会打开预置的操作教程，跟着这个教程一步一步慢慢体会emacs和一般文本编辑器的不同吧。

SBCL

一度成为common lisp在linux最流行实现，其全称为Steel Bank Common Lisp。为什么叫这个奇怪的名字？因为它起源于Carnegie Mellon University Common Lisp，作者William Newman最早在1999年宣布SBCL是CMUCL的改版，于是恶趣味的参考Carnegie Mellon University Common Lisp，将新的实现命名为Steel Bank Common Lisp——Carnegie和Mellon是分别靠钢铁业和银行业发家致富的。

更适合大众的口味，sublime text作为文本编辑器

之前是不是觉得不太习惯emacs这个编辑器，不要紧，sublime text也是一个选择。sublime text是一款非常流行也非常性感的文本编辑器，关于安装和插件调整网上有很多资料，需要的朋友可以自己搜索(先看看官网)，这里只说一说它对cl的支持。

  1. 在上一节中你已经做好了除去安装emacs之外的所有步骤。
  2. 确保你的sublime text已经安装并可以正常使用(sublime text 3)。
  3. 确保Package Control部件工作正常。
  4. 安装SublimeREPL()：
    在sublime text按下Ctrl+Shift+p呼出控制框，输入install packages；
    跳转后输入SublimeREPL，回车就可以进行安装；
    通过选项Tools->SublimeREPL->common lisp->SBCL就可以在subl中sbcl了
